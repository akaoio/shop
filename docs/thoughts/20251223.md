- Việc hash tỉ mỉ từng file json sau build đã xong
- Các file hash này dùng để phân biệt phiên bản của từng file json
- Khi hash thay đổi, tức là nội dung json cần được fetch mới
- Việc cần xử lý trước tiên là locale

- Lưu hash vào đâu trong browser? -> Ta lưu hash vào indexeddb

- Khi browser bật, nó lấy dữ liệu từ indexeddb ra trước -> sau đó nó truy vấn hash xem có trùng hash cũ trong indexeddb không?
  - Nếu không trùng -> fetch -> lưu vào indexeddb
  - Nếu trùng -> bỏ qua và sử dụng luôn data
